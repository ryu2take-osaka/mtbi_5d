# Project: MBTI 5D Visualizer (vNext++ Final Specification)

## 1. 概要

MBTIの5指標（E-I, S-N, T-F, J-P, A-T）を、3次元空間上の座標・発光・密度・粒子フロー、および動的な「補完ビーム」によって可視化するWebアプリケーション。
「性格を空間に置き、関係性を時間で理解させるエンジン」として、自身の性格や他者との相性を視覚的・体験的に直感できるよう設計されている。

---

## 2. ユーザーインターフェース (UI/UX)

### 2.1 状態遷移モデル（2段階モデル）
入力中のノイズを排除し、解析体験に集中させるため以下のプロセスを持つ。
1. **EDITING (編集)**: スライダー操作中。球体のみ即時更新。ビーム・ナレーションは非表示。
2. **ANALYZING (解析中)**: 操作停止から約600ms後に移行（**P1とP2の両方に入力がある場合のみ実行**）。解析演出（チャージ効果）を表示。URLパラメータ経由で初期ロードされた際も、両者の入力があれば即座にこのモードからスタートする。
3. **PLAYING (再生)**: 解析完了後、タイムラインに沿ってビームとナレーションを順次表示（20秒サイクル）。
4. **IDLE (待機)**: ユーザー入力が一方のみ、または再生完了後。相手（P2）が未入力（オール50）の初期状態では、P2のプロット（球体とガイドライン）は非表示となる。

### 2.2 スライダー設計（16Personalities準拠）
- **左右反転ロジック**: 内部値 0〜100 に対して、50以下の場合は左側の特性、50以上の場合は右側の特性として表示。
  - 例：思考型(T)が内部値42の場合、UI上は「58% 思考型」と表示し、ラベルを太字化する。
- **タブシステム**: P1 / P2 / SCORE の3タブ等幅グリッド構成。
  - **P1 / P2**: 各ユーザーの性格設定。メインラベルの下に小さくタイプ名（ENTJ等）を併記。
  - **SCORE**: 各種アクションと解析結果を表示するサマリーパネル。
- **微調整ボタン**: スライダーの両端に配置。モバイル操作性を考慮したコンパクトサイズ（28px）。

### 2.3 画面レイアウト（vNext+++）
- **三分割構造**: `[P1 Stack] [ 3D Canvas ] [P2 Stack]`
  - 左右に各ユーザーの機能スタック（Side Stack）を常時表示。
  - 中央に性格の座標を示す3D空間を配置。
- **モバイル対応レイアウト**:
  - 画面幅の狭いスマートフォン等では、Side Stackが3Dキャンバスの上面（オーバーレイ）の左右端に配置され、描画空間を最大化しつつ被りを防ぐ。
  - SCOREタブの距離・補完スコア表示も横並びを維持しつつ、ノードの適切な縮小によりガタつきを防止する。
  - **ナレーション演出**: モバイル環境ではフォントサイズを18pxに縮小し、左右に適切なパディング（40px）を設けることで、可読性の向上と画面端での不自然な折り返しを防止する。
- **コントロールパネル**: 下部固定。スコアサマリーとアクションボタンを表示。
- **スクロール用余白（Gutter）**: 左右に40pxの余白を確保。

### 2.4 機能スタックのLVスペクトラム表示
各認知機能ノードの下に `LV:0〜100` を表示し、スライダー値に連動してリアルタイムで変化する。
- **計算式**: `LV[i] = posWeight[i] × ((attScore + domScore) / 2)`
  - `posWeight`: スタック位置の重みづけ [1.0, 0.75, 0.5, 0.25]（主機能→劣等機能）
  - `attScore`: 態度スコア（内向機能は EI が高いほど、外向機能は EI が低いほど高くなる）
  - `domScore`: 領域スコア（N は NS が低いほど、S は NS が高いほど、T は TF が低いほど、F は TF が高いほど高くなる）

## 3. 3次元空間マッピング

### 3.1 座標軸 (Coordinate System)
- **空間構造**: 床・背面・側面の3面で構成される「コーナー」を基準とする。
- **厚みのある軸**: 視認性を高めるため円柱（CylinderGeometry）による太い軸を採用。
  - **X軸 (N-S)**: 黄色 (#E4AE3A)
  - **Y軸 (T-F)**: 緑色 (#33A474)
  - **Z軸 (J-P)**: 紫色 (#8A61A6)

### 3.2 補助線と個体
- **個体（球体）**: 発光（E-I）、サイズ（A-T）、揺れ（Turbulence）で個性を表現。
- **補助線 (Guide Lines)**: 各球体から壁・床に向かって伸びる指標カラーの破線。

## 4. ビジュアル表現（補完ビーム）

関係性の主役として、二人のエネルギーのやり取りを可視化する。

### 4.1 補完ビームの種類（優先順位順）

| 優先順位 | タイプ | 発動条件 | ビーム色 | 意味 |
|---|---|---|---|---|
| 1 | **Resonance (共鳴)** | 同じスタック位置、同じ機能（例: P1[1]=Ne ↔ P2[1]=Ne） | マゼンタ | 最強の同質共鳴。同じ認知モードを極める |
| 2 | **Mirror (鏡)** | 同じスタック位置、同ドメイン・逆態度（例: P1[1]=Ne ↔ P2[1]=Ni） | シアン | 同じ次元を内・外から見る鏡関係 |
| 3 | **Rescue (救済)** | 異なるスタック位置、同じ機能（例: P1[1]=Ti ↔ P2[3]=Ti） | ゴールド | 一方の強みが他方の弱点を補う |
| 4 | **Expansion (拡張)** | スタック全体の知覚軸または判断軸が異なる | グリーン | 全体的な視野の広がりと相乗効果（3Dビームなし。スコアのみ加算） |

> **Mirrorの重要条件**: 必ず**同じスタック位置**（i=j）であること。位置が異なる場合はRescueまたはExpansionとして扱う。MirrorとExpansionの違いは「位置揃いの逆態度（Mirror）」か「スタック全体の軸差（Expansion）」か。

### 4.2 ビジュアルパラメータ（シンボリック表示）
- **表示制限**: 最大2本（最優先のコア補完 + サブ1本）。EXPANSION はスコア計算のみでビームは表示しない。
- **優先順位の決定**: priority フィールド（1=最高）でまず分類。同 priority 内はスコア `s` の降順。
- **UI発光連動**: タイムラインに同期し、サイドの機能スタックUIが発光（Core=強, Sub=中, Mirror=パルス）。
- **最適化**: CylinderGeometryのセグメントを削減(8)し、モバイルでも安定動作。

### 4.3 アニメーションと粒子
- **パルス（脈動）**: 強度 `S` に応じた周期的な太さの変化（呼吸感）。
- **粒子統合**: 粒子はビームの経路上を流れる。始点で生成され、経路に沿って移動し、終点でフェードアウト。

## 5. ロジックと機能

### 5.1 補完スコア計算式（双方向・統合モデル）

#### 位置重み係数 (weights)
スタック位置に応じた加重係数。上位ポジションほど相互作用の影響が大きい。

| 位置 | 機能種別 | 強度 (strength) | 重み (weight) |
|---|---|---|---|
| 1 | 主機能 (Hero) | 1.0 | 1.3 |
| 2 | 補助機能 (Parent) | 0.8 | 1.1 |
| 3 | 第三機能 (Child) | 0.6 | 0.9 |
| 4 | 劣等機能 (Inferior) | 0.4 | 0.7 |

#### Rescue スコア計算式
```js
// AがBより上位ポジションで同機能を持つ場合
S(A→B) = (1 - B_strength) * A_strength * A_weight
S(B→A) = (1 - A_strength) * B_strength * B_weight
Total_Rescue = S(A→B) + S(B→A)
```

**具体例: ENTPのNe(pos=1) vs INTPのNe(pos=2)**
```
S(A→B) = (1 - 0.8) * 1.0 * 1.3 = 0.26
S(B→A) = (1 - 1.0) * 0.8 * 1.1 = 0.00
Total = 0.26  → ENTP が INTP の Neを上位から支える一方向の救済
```

#### Mirror/Resonance スコア
```js
Resonance_s = strength[i] * 1.5 * weight[i]   // 最大: 1.0 * 1.5 * 1.3 = 1.95
Mirror_s    = strength[i] * 0.9 * weight[i]   // 最大: 1.0 * 0.9 * 1.3 = 1.17
```

#### Expansion スコア（スタック全体）
```
知覚軸が異なる（NiSe ↔ NeSi）: +0.55
判断軸が異なる（TiFe ↔ TeFi）: +0.45
```

#### Complement Score の集計
```js
totalS = sum(all complementData[i].s)
compScore = min(100, totalS * 14)   // スケール係数14で最大≈100に正規化
```

### 5.2 20秒タイムラインサイクル
1. **導入 (0-2s)**: 関係性の断定とタイプ表示。
2. **コア補完 (2-6s)**: 最強のメインビーム表示と解説。
3. **サブ補完 (6-12s)**: 上位の補助的ビームを順次紹介。
4. **差異 (12-16s)**: ズレ要因や注意点の提示。
5. **俯瞰 (16-20s)**: 総括と全体バランスの提示。
