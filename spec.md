# Project: MBTI 5D Visualizer (vNext++ Final Specification)

## 1. コンセプト
**「性格を空間に置き、関係性を時間で理解させるエンジン」**
- **性格＝空間内の存在**: 座標・発光・サイズで個性を表現。
- **関係性＝エネルギーの補完**: 補完ビームによる動的な相互作用。

---

## 2. ユーザーインターフェース (UI/UX)

### 2.1 状態遷移モデル
入力中のノイズを排除し、解析体験に集中させるため2段階の状態を持つ。
1. **EDITING (編集)**: スライダー操作中。球体のみ即時更新。ビーム・ナレーションは非表示。
2. **ANALYZING (解析中)**: 操作停止から約600ms後に移行。解析演出（チャージ効果）を表示。
3. **PLAYING (再生)**: 解析完了後、タイムラインに沿ってビームとナレーションを順次表示。
4. **IDLE (待機)**: 全ての再生が完了した俯瞰状態。

### 2.2 ナレーションシステム
画面中央または専用パネルに、再生フェーズと同期した解説テキストを表示。
機能略称（Ti, Ne等）には必ず日本語の説明（内向思考：論理的探究等）を付記。

---

## 3. 補完ビーム設計 (Complement Beams)

ビームは二人の間の「エネルギーの不足を補う流れ」として定義される。

### 3.1 強度計算式
各機能 $f$ について、供給側の強さを $A[f]$、受給側の不足度を $Need_B[f] = 1 - B[f]$ とする。
- **基本補完強度**: $S(A \to B, f) = Need_B[f] \times A[f]$
- **スタック補正**: $S' = S \times p(f)$
  - 順位重み $p(f)$: 1位:1.3 / 2位:1.1 / 3位:0.9 / 4位:0.7

### 3.2 ビームの種類
1. **Rescue (救済型)**: 同一機能（例: P1のTi → P2のTi）。不足を直接埋める。金色ビーム。片方向。
2. **Mirror (鏡型)**: 逆属性機能（例: Ti ↔ Te）。視点を広げ合う。シアンビーム。双方向（往復）。
3. **Expansion (拡張型)**: 異種機能。相乗効果。拡散・波紋エフェクト。

### 3.3 ビジュアルパラメータ
- **太さ**: $0.01 + 0.07 \times S^{1.8}$
- **透明度**: $0.3 + 0.7 \times S$
- **E/I補正**: 両者の外向性が高いほどビームが太く、エネルギッシュになる。

## 4. 20秒タイムラインサイクル

空間での同時多発的な情報を避け、時間軸で物語を語る。

1. **導入 (0-2s)**: 全体俯瞰、二人のタイプ名表示、関係性を一言で定義。
2. **コア補完 (2-6s)**: 最も強い1〜2本のメインビームを表示。太く安定した描画。
3. **サブ補完 (6-12s)**: 上位2〜4本の補助的ビームを順次表示。テンポ良く切り替える。
4. **差異 (12-16s)**: 補完の弱い部分、ズレをノイズやビームの途切れで表現。
5. **俯瞰 (16-20s)**: 主要ビームを全て弱く表示し、総合評価を提示。

## 5. 理論定義（認知機能日本語訳）

| 略称 | 日本語名称 | 説明 |
| --- | --- | --- |
| **Ni** | 内向的直観 | 本質を見抜き、長期的な展望を描く |
| **Ne** | 外向直観 | 多様な可能性を広げ、アイデアを繋ぐ |
| **Si** | 内向感覚 | 過去の経験を記憶し、安定を保つ |
| **Se** | 外向感覚 | 今この瞬間の刺激を捉え、行動する |
| **Ti** | 内向思考 | 内部的な論理を追求し、仕組みを解く |
| **Te** | 外向思考 | 外部の効率を求め、目標を達成する |
| **Fi** | 内向感情 | 自身の価値観に誠実であり、調和を重んじる |
| **Fe** | 外向感情 | 周囲の感情に同調し、人間関係を円滑にする |

## 6. 技術実装
- **Three.js**: ビームは `TubeGeometry` または `Line2` を使用して厚みを表現。
- **State Machine**: 解析フェーズの状態管理。
- **Narration Engine**: タイムラインに同期したテキスト生成と表示。
