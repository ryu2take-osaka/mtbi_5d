# Project: MBTI 5D Visualizer (vNext++ Final Specification)

## 1. 概要

MBTIの5指標（E-I, S-N, T-F, J-P, A-T）を、3次元空間上の座標・発光・密度・粒子フロー、および動的な「補完ビーム」によって可視化するWebアプリケーション。
「性格を空間に置き、関係性を時間で理解させるエンジン」として、自身の性格や他者との相性を視覚的・体験的に直感できるよう設計されている。

---

## 2. ユーザーインターフェース (UI/UX)

### 2.1 状態遷移モデル（2段階モデル）
入力中のノイズを排除し、解析体験に集中させるため以下のプロセスを持つ。
1. **EDITING (編集)**: スライダー操作中。球体のみ即時更新。ビーム・ナレーションは非表示。
2. **ANALYZING (解析中)**: 操作停止から約600ms後に移行（**P1とP2の両方に入力がある場合のみ実行**）。解析演出（チャージ効果）を表示。URLパラメータ経由で初期ロードされた際ও、両者の入力があれば即座にこのモードからスタートする。
3. **PLAYING (再生)**: 解析完了後、タイムラインに沿ってビームとナレーションを順次表示（20秒サイクル）。
4. **IDLE (待機)**: ユーザー入力が一方のみ、または再生完了後。相手（P2）が未入力（オール50）の初期状態では、P2のプロット（球体とガイドライン）は非表示となる。

### 2.2 スライダー設計（16Personalities準拠）
- **左右反転ロジック**: 内部値 0〜100 に対して、50以下の場合は左側の特性、50以上の場合は右側の特性として表示。
  - 例：思考型(T)が内部値42の場合、UI上は「58% 思考型」と表示し、ラベルを太字化する。
- **タブシステム**: P1 / P2 / SCORE の3タブ等幅グリッド構成。
  - **P1 / P2**: 各ユーザーの性格設定。メインラベルの下に小さくタイプ名（ENTJ等）を併記。
  - **SCORE**: 各種アクションと解析結果を表示するサマリーパネル。
- **微調整ボタン**: スライダーの両端に配置。モバイル操作性を考慮したコンパクトサイズ（28px）。

### 2.3 画面レイアウト（vNext+++）
- **三分割構造**: `[P1 Stack] [ 3D Canvas ] [P2 Stack]`
  - 左右に各ユーザーの機能スタック（Side Stack）を常時表示。
  - **水位表現 (Water Level)**: 各機能ノード内部にLV（強さ）を水位として塗りで可視化。下から上へ充填される水平ラインにより、直感的な強弱把握を可能にする。
- **モバイル対応レイアウト**:
  - 画面幅の狭いスマートフォン等では、Side Stackが3Dキャンバスの上面（オーバーレイ）の左右端に配置される。
  - ナレーション演出はフォントサイズ18px、左右パディング40pxで可読性を確保。
- **コントロールパネル**: 下部固定。スコアサマリーとアクションボタンを表示。
- **スクロール用余白（Gutter）**: 左右に40pxの余白を確保。

## 3. 3次元空間マッピング

### 3.1 座標軸 (Coordinate System)
- **空間構造**: 床・背面・側面の3面で構成される「コーナー」を基準とする。
- **厚みのある軸**: 視認性を高めるため円柱（CylinderGeometry）による太い軸を採用。
  - **X軸 (N-S)**: 黄色 (#E4AE3A)
  - **Y軸 (T-F)**: 緑色 (#33A474)
  - **Z軸 (J-P)**: 紫色 (#8A61A6)

### 3.2 補助線と個体
- **個体（球体）**: 発光（E-I）、サイズ（A-T）、揺れ（Turbulence）で個性を表現。
- **補助線 (Guide Lines)**: 各球体から壁・床に向かって伸びる指標カラーの破線。

## 4. ビジュアル表現（軽量化・意味優先）

描画負荷を最小化し、関係性の「意味」を一目で伝える静的表現を採用。

### 4.1 補完ビームの種類（静的表現）
1. **Rescue (救済型)**: 金色の直線。相手のスタックを補完する接続を示す。
2. **Mirror (鏡型)**: シアンの直線。属性反転機能（Ni ↔ Ne等）による視点の拡張を示す。

### 4.2 表現ルール
- **静的化**: 常時アニメーション、発光の脈動、粒子の流動表現をすべて廃止。
- **視認性**: シリンダー（CylinderGeometry）のセグメントを削減(6)し、適度な透明度(0.6)で固定。
- **水位連動**: 各機能ノード内の水位（塗り）が強さを示し、ビームがその関係性を示す。
- **Mirrorアイコン**: スコア欄に `🔁` アイコンを表示し、係数補正が適用されていることを明示。

## 5. ロジックと機能

### 5.1 補完スコア（Mirror係数化）
補完の「質」を評価するため、基礎補完量に噛み合わせやすさを乗算する。

**式: $S_{total} = S_{rescue} \times (1 + Mirror_{factor})$**

1. **Rescueスコア (基礎点)**
   - P1(1) ↔ P2(4) の完全補完: +40点
   - P1(2) ↔ P2(4) または P1(1) ↔ P2(3) の部分補完: +25点
   - 上記を正規化（最大100）。

2. **Mirror係数 (倍率)**
   - 同一ポジションでのMirror関係: +0.04
   - 上位スタック同士のMirror関係: +0.04
   - 機能LV（水位）の差が小さい(<20): +0.02
   - **上限**: 係数合計は最大 0.12 まで。

### 5.2 機能レベル (LV) 算出
- 認知機能の強さを 0〜100 で算出。各個人の性格パラメータからHarold Grantスタックに基づき導出。
- 水位表現のパラメータとして利用。

### 5.3 20秒タイムラインサイクル
1. **導入 (0-2s)**: 関係性の断定とタイプ表示。
2. **コア補完 (2-6s)**: 最強のメインビーム表示と解説。
3. **サブ補完 (6-12s)**: 上位の補助的ビームを順次紹介。
4. **差異 (12-16s)**: ズレ要因や注意点の提示。
5. **俯瞰 (16-20s)**: 総括と全体バランスの提示。

### 5.3 理論定義（認知機能日本語訳）
Ni：内向的直観 / Ne：外向直観 / Si：内向感覚 / Se：外向感覚 / Ti：内向思考 / Te：外向思考 / Fi：内向感情 / Fe：外向感情
