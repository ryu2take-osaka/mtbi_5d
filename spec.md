# Project: MBTI 5D Visualizer (Final Specification)

## 1. 概要

MBTIの5指標（E-I, S-N, T-F, J-P, A-T）を、3次元空間上の座標・発光・密度・粒子フローによって可視化するWebアプリケーション。
「性格＝エネルギーの動的な振る舞い」として表現し、ユーザーが自身の性格や他者との相性を視覚的・体験的に直感できるよう設計されている。

---

## 2. ユーザーインターフェース (UI/UX)

### スライダー設計（16Personalities準拠）
- **左右反転ロジック**: 内部値 0〜100 に対して、50以下の場合は左側の特性、50以上の場合は右側の特性として表示。
  - 例：思考型(T)が内部値42の場合、UI上は「58% 思考型」と表示し、ラベルを太字化する。
- **タブシステム**: P1 / P2 / SCORE の3タブ等幅グリッド構成。
  - **P1 / P2**: 各ユーザーの性格設定。メインラベルの下に小さくタイプ名（ENTJ等）を併記し、高さを抑制。
  - **SCORE**: 各種アクションと結果表示を集約したサマリーパネル。
- **微調整ボタン**: スライダーの両端に配置。モバイル環境では指での操作性を考慮しつつ、28pxのコンパクトサイズでUIの圧迫感を低減。

### 画面レイアウト
- **Canvasエリア**: 画面上部の広大な3D空間。ビジュアルを最大限際立たせるため、パネルをコンパクト化。
- **コントロールパネル**: 下部固定。モバイルでは最大高さ（55〜60vh）と独立スクロールを設け、**左右に40pxの「スクロール用余白（Gutter）」**を確保することで、スライダー誤操作を防ぎつつ快適なスクロールを可能にした。
- **下部余白**: 各種ブラウザの操作バーによる隠蔽を防ぐため、スクロール最下部に十分なパディング（120px）を確保。
- **SCOREパネル**: 距離スコア、補完スコア、相性ランク、機能スタック可視化、SNS共有、画像保存、URLコピー、および両者の性格要約を簡潔に表示。

## 3. 3次元空間マッピング

### 座標軸 (Coordinate System)
- **空間構造**: 床・背面・側面の3面で構成される「コーナー」を基準とする。
- **厚みのある軸**: 標準の1px線ではなく、視認性を高めるため円柱（CylinderGeometry）による太い3色の軸を採用。
  - **X軸 (N-S)**: 黄色 (#E4AE3A)
  - **Y軸 (T-F)**: 緑色 (#33A474)
  - **Z軸 (J-P)**: 紫色 (#8A61A6)

### 補助線 (Guide Lines)
- 各球体からそれぞれの壁・床に向かって、指標カラーに対応した**破線（Dashed Line）**が伸び、空間内での正確な位置を把握可能にする。

## 4. ビジュアル表現（エネルギー）

### 球体（Entity）
- **発光 (E-I)**: 外向的(E)であるほど発光強度とBloom効果が増加。
- **サイズ (A-T)**: 
  - 自己主張型(A): 最大130%のサイズ（硬さ・安定感の表現）。
  - 慎重型(T): 最小70%のサイズ（ブレ・繊細さの表現）。
- **挙動 (Turbulence)**: 慎重型(T)に傾くほど、上下（垂直方向）にゆっくりと揺れる。
  - 振幅: A-Tが50%（ニュートラル）時の球体サイズを超えない範囲で振動。
  - 周期: 1秒で上限から下限へ移動（1Hz）。
- **タイプカラー**: NT/NF/SJ/SPの分類に基づく配色。

### 粒子フロー (Energy Particle Flow)
二人の性格エネルギーの相互作用を可視化する。
- **方向**: より外向的（E値が高い/内部値が0に近い）な個体から、より内向的（I値が高い/内部値が100に近い）な個体へ流れる。
- **速度**: 二人のEI値の差（勾配）に比例して加速する。
- **密度と揺れ**: 二人の合計エネルギー（外向性の総和）が高いほど、粒子の数が増え、不規則な振動（Jitter）が激しくなる。

## 5. ロジックと機能

### 相性算出
- **計算式**: `Score = (1.0 - (Euclidean Distance / 3.0)) * 100`
- **評価基準**: 
  - S (80〜): 共鳴
  - A (60〜): 高相性
  - B (40〜): 中間
  - C (20〜): 差異
  - D (0〜): 対立

### 補完機能スタックスコア (Function Complement Score)
従来の距離ベース相性（思想的な共鳴）に加え、お互いの弱点を補い支え合う「機能的な補完関係」を評価する。

#### 補完の種類
1. **Rescue Complement（救済型補完）**
   - **定義**: 同じ認知機能で、相手の劣等機能（4位）を自分の強い機能で直接フォローする関係。
   - **最強例**: P1の(1) Dominant と P2の(4) Inferior が同じ機能。
   - **特徴**: 相手の致命的な弱点を自分が日常的に得意として支える → 「支え合い」感が非常に強い。
   - **オーラ**: 金色・最大glow + 粒子エフェクト（最高優先）。
2. **Mirror Complement（鏡型補完）**
   - **定義**: 同じスタック位置（例: 両方2位）でありながら、機能の向き（外向/内向）が逆の関係（Ne↔Ni、Te↔Ti、Se↔Siなど）。
   - **特徴**: 同じ役割で逆の視点を提供し合い、相乗効果を生む。視野を広げ合い、刺激し、成長を促進し合う関係。
   - **オーラ**: 緑〜シアン系の中glow（Rescueの金/青とは明確に色を分ける）。

#### 加点テーブル
該当するすべての補完を合算して算出する。

| 補完種類          | 条件例                              | 加点 | オーラ強度                  |
|-------------------|-------------------------------------|------|-----------------------------|
| Rescue            | P1(1) ↔ P2(4) 同じ機能             | +40  | 最高（金・最大glow+粒子）   |
| Rescue            | P1(1) ↔ P2(3) 同じ機能             | +25  | 強い（青・強glow）          |
| Rescue            | P1(2) ↔ P2(4) 同じ機能             | +20  | 強い（青・強glow）          |
| Rescue / Mild     | P1(1) ↔ P2(2) など                 | +15  | 中（緑・中glow）            |
| **Mirror**        | **同じ位置で逆属性**（Ne↔Niなど） | **+10** | 中（緑〜シアン・中glow）   |
| Mild              | P1(2) ↔ P2(3) など                 | +12  | 中                          |
| その他            | -                                   | +0〜5| なし or 微弱               |

※ **軸の相補性ボーナス**: 知覚軸反対・判断軸反対がペアで成立する場合、各+15点。Mirror補完とも併用可能。

#### 合計スコアランク
- 80〜100：**S+ Complement**（最高の補完関係・支え合い）
- 60〜79：**A**
- 40〜59：**B**
- 20〜39：**C**
- 0〜19：**D**

#### 可視化（Aura Effect）
- **金色オーラ**: Rescue最強パターン（1-4）。
- **青色オーラ**: その他の強いRescue補完。
- **緑〜シアン色オーラ**: Mirror補完およびMild補完。
- **粒子パルス**: 二人のエネルギーが共鳴している状態を表現。

### 共有と保存
- **URL共有**: 二人の状態をバイナリ化し、Base64エンコードしてURLパラメータ（`?d=...`）に付与。
- **URLをコピー**: 現在の設定URLをワンクリックでクリップボードにコピー（トースト通知付き）。
- **X共有**: 現在の結果と復元用URLを自動的にポストする画面へ遷移。
- **スクリーンショット**: 3D表示エリアをPNG形式で保存。ファイル名に両者のタイプ名を自動付与。
