# Project: MBTI 5D Visualizer (Refactored Full Spec)

## 1. 概要

MBTIの5指標（E-I, S-N, T-F, J-P, A-T）を、3次元空間および「認知機能スタック」の連動によって可視化するWebエンジン。
単なる相性診断に留まらず、個々の認知機能が「A/T（安定/揺れ）」や「LV（使用頻度）」によってどのように振る舞うかを詳細に解き明かす。

---

## 2. ユーザーインターフェース (UI/UX)

### 2.1 動作モード

1. **EDITING (編集)**: スライダー操作中。球体のみ即時更新。
2. **PLAYING (再生)**: 解析完了後、以下のいずれかのモードで動作：

   * **Compatibility (相性モード)**: 20秒サイクルで相性と補完点を解説。
   * **Individual Description (個別解説モード)**: 選択したユーザーの各認知機能を解説し、次に**スタック特性（Expanded/Inverted/Tilted/Balanced）の総評**を行い、最後に「二つ名（Epithet）」と「通称」を公開。

---

## 3. 解説・ナレーションシステム

### 3.1 称号（二つ名）決定ロジック

個別の機能解説の最後に、タイプとA/T、および機能の使い方の偏り（Inverted）に基づいて称号を授与する。

### 3.2 スタック特性（総評）ロジック

個別の機能解説の後に、全機能のバランスから4つの状態を判定する。

---

## 4. 演出とビジュアル (Visual Effects)

### 4.1 輝度演出 (Glow & Bloom)

* カラーグループ: NT（紫）、NF（緑）、SP（黄）、SJ（青）。

## 6. スコアリング仕様（具体化）

### 6.1 MBTIタイプ文字列生成

5軸スライダー値（0〜100）からタイプ文字列を生成する。

* `EI < 50 = E` / `> 50 = I`
* `NS < 50 = N` / `> 50 = S`
* `TF < 50 = T` / `> 50 = F`
* `JP < 50 = J` / `> 50 = P`
* `AT < 50 = A` / `> 50 = T`
* `=50` の場合は曖昧値として `X`

例: `INTP-T`, `ENFJ-A`

### 6.2 認知機能LV（使用強度）計算

各タイプに定義された4機能スタックを対象に、機能ごとの使用強度 `LV` を算出する。

#### ポジション補正係数

* 主機能: `1.00`
* 補助機能: `0.75`
* 第3機能: `0.50`
* 劣等機能: `0.25`

#### 軸反映

各機能は以下2要素の平均で基礎値を作る。

* 内外向（i/e）適合度
* 領域（N/S/T/F）適合度

#### 計算式

`rawLevel = positionWeight × ((attScore + domScore) / 2)`

最終的に `0〜100` に丸め込み整数化する。


### 6.3 Rescue Score（補完点）

相手の劣等機能・第3機能を、自分の主機能/補助機能で支えられる場合に加点する。

#### 基本ルール

* 自分の主機能 = 相手の劣等機能 → FULL加点
* 自分の補助機能 = 相手の劣等機能 → PARTIAL加点
* 自分の主機能 = 相手の第3機能 → PARTIAL加点

双方A/Bに対して対称計算し、合算する。

### 6.4 Mirror Factor（鏡像係数）

同じ機能軸で、内向/外向のみ反転している場合に共鳴値を加算する。

例:

* `Ti ↔ Te`
* `Ni ↔ Ne`
* `Fi ↔ Fe`
* `Si ↔ Se`

#### 加点条件

* 同順位同士で一致: `position`
* 両者とも上位2機能: `strong`
* LV差が20未満: `lvNear`

総和に上限 `cap` を設定する。

### 6.5 スタック状態判定

機能LVを `f1,f2,f3,f4` とする。

* **expanded**: `f4 >= 10`
* **inverted**: `f3 > f2 + 5`
* **tilted**: `|f2-f3| <= 5`
* **balanced**: その他

### 6.6 A-T補正（エネルギードリフト）
A/T（Assertive/Turbulent）は関係性に生まれる変化エネルギーや刺激量として扱い、総合的な「補完度」に対して±10点の補正を行う。

#### 計算式
`modifier = round(((t1 + t2) / 10) - 10)`
※ `t` は各人のTurbulent率（0〜100）

#### 解釈
* **マイナス補正（A同士）**: 安定・完成・変化少なめ
* **プラス補正（T同士）**: 成長圧・刺激・推進力大
* **ゼロ付近（混合）**: 安定と刺激のバランス型

### 6.7 共鳴コード（Resonance Codes）
「補完度」と「類似度」のランク（S/A/B/C/D）を組み合わせた25パターンの診断結果。ナレーションの最終フェーズ（OVERVIEW）およびSCOREパネルに表示される。

#### コード構造
`[補完度ランク] + [類似度ランク]` （例：SA, BC）
各コードには独自の「タイトル」と「解説文」が定義されている。
